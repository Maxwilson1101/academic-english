# Report Genre Specification

## Scope

Report genres construct and store scientific or technical knowledge as a static account of how entities, categories, or systems are organized in the world.

Use a report genre when the task requires explanation of stable knowledge rather than recounting events, arguing a position, giving instructions, or telling a story.

For language templates that realize these structures, use [Report Expression Patterns](report-expressions.md).

## General Rules

- Keep the text objective, informational, and taxonomy-oriented.
- Organize content by semantic relations, not chronology.
- Use present-tense generalizations unless the subject requires another tense.
- Begin by identifying the entity, class, classification basis, or whole being reported.
- Make stage boundaries clear through paragraphing, headings, topic sentences, or another visible structure.
- Treat each major characteristic, type, component, function, or activity as a distinct phase when the text is long enough to require structure.
- Do not include personal opinion, persuasion, narrative suspense, or procedural commands unless required by the source task.

## Genre Types

### Descriptive Report

**Purpose:** Describe the characteristics of one class of entity.

**Use when:** The target is a single entity class, physical or abstract, and the task is to describe what it is like.

**Required stages:**

1. **Classification**
   - Identify the entity to be described.
   - Situate the entity in a relevant taxonomy or category.
   - State aliases or defining identity information when useful.

2. **Description**
   - Describe selected characteristics of the entity.
   - Possible characteristic phases include appearance, attributes, behavior, functions, diet, habitat, status, origin, design features, or other relevant properties.
   - Each phase should focus on one coherent characteristic set.
   - Use [descriptive report expressions](report-expressions.md#descriptive-report-expressions) to express position, structure, shape, size, weight, function, and property when relevant.

**Structure rules:**

- The Classification stage must precede the Description stage.
- Description phases are variable and should be selected according to the entity and communicative purpose.
- Do not classify multiple peer types as the main organizing principle; use a classifying report for that.
- Do not decompose the entity into parts as the main organizing principle; use a compositional report for that.

### Classifying Report

**Purpose:** Classify members of a general class into types according to explicit criteria.

**Use when:** The target is a general class with multiple subtypes, and the task is to explain how those subtypes are distinguished.

**Required stages:**

1. **Classification System**
   - State the general class being classified.
   - State the classification criterion or criteria.
   - Name the resulting types.
   - Acknowledge that different criteria can produce different classification systems when relevant.
   - Use [classifying report expressions](report-expressions.md#classifying-report-expressions) to name types and state classification criteria.

2. **Types**
   - Describe each type identified in the classification system.
   - Provide distinguishing characteristics for each type.
   - Include subtypes of a type when they are necessary to complete the classification.

**Structure rules:**

- The Classification System stage must precede the Types stage.
- Use the same classification basis consistently within one report.
- Present each main type in a separate paragraph or clearly marked phase when there is enough detail.
- Begin each main type phase with a topic sentence that identifies the type.
- Do not mix unrelated classification criteria unless the report explicitly compares alternative systems.

### Compositional Report

**Purpose:** Describe a whole by identifying its parts and explaining their roles, functions, or activities.

**Use when:** The target is a whole entity, system, structure, group, or environment, and the task is to explain what it is composed of.

**Required stages:**

1. **Classification**
   - Identify and classify the whole entity to be decomposed.
   - Establish the whole as the organizing focus of the report.

2. **Components**
   - Identify the components of the whole.
   - Describe each component's function, role, or activity.
   - Present each major component as a distinct phase when the text contains multiple components.
   - Use [compositional report expressions](report-expressions.md#compositional-report-expressions) to state whole-part relations.

**Optional stage:**

3. **Technical Definition**
   - Distill the preceding component information into a concise definition of the whole.
   - Use only when a final synthesized definition improves clarity.

**Structure rules:**

- The Classification stage must precede the Components stage.
- The Components stage must use a part-whole relation as its organizing principle.
- Acceptable component presentation patterns include component-function and component-activity.
- Do not organize primarily by subtype distinctions; use a classifying report for that.
- Do not organize primarily by general characteristics without part-whole relations; use a descriptive report for that.

## Genre Selection

- Use **descriptive report** for one entity class and its characteristics.
- Use **classifying report** for one general class divided into types by criteria.
- Use **compositional report** for one whole divided into components.
- If a topic supports more than one relation, choose the genre matching the primary task:
  - characteristics -> descriptive report
  - types -> classifying report
  - parts -> compositional report

## Output Contract

A valid report must:

- Declare or imply exactly one primary organizing relation: characteristics, types, or parts.
- Include all required stages for the selected genre.
- Maintain consistent terminology for the target entity, class, types, or components.
- Use phases to separate major semantic units.
- Provide enough detail for each phase to explain its relevance to the report's organizing relation.
- Use expression patterns from [Report Expression Patterns](report-expressions.md) only when they match the selected genre and organizing relation.

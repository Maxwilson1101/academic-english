# Report Expression Patterns

## Scope

Report expression patterns provide reusable wording for realizing common meanings in report genres.

Use this file when generating or revising language inside a report. For genre purposes, required stages, and selection rules, use [Report Genre Specification](report.md).

Replace `A`, `B`, `C`, `X`, `Y`, `Z`, `N`, `unit`, `criterion`, `class`, `type`, `component`, and `function` with topic-specific content.

## General Rules

- Select expressions according to the report's primary organizing relation: characteristics, types, or parts.
- Use descriptive expressions for characteristics of one entity class.
- Use classifying expressions only when naming types under a stated classification basis.
- Use compositional expressions only when naming parts of a whole.
- Use parallel expression forms when listing comparable characteristics, types, or components.
- Keep placeholders semantically consistent: do not use type expressions for components or component expressions for types.

## Classifying Report Expressions

Use these patterns in the Classification System stage of a [classifying report](report.md#classifying-report).

- `There are N types of class: type and type.`
- `There are N kinds of class: type and type.`
- `There are N classes of class: type and type.`
- `There are N categories of class: type and type.`
- `There are N varieties of class: type and type.`
- `Class consists of N types: type and type.`
- `Class comprises N kinds: type and type.`
- `Class can be classified into N categories: type and type.`
- `Class can be divided into N varieties: type and type.`
- `Class is classified according to criterion.`
- `Class is divided on the basis of criterion.`
- `Class is classified depending on criterion.`

## Compositional Report Expressions

Use these patterns in the Components stage of a [compositional report](report.md#compositional-report).

- `X consists of Y and Z.`
- `X is made up of Y and Z.`
- `X can be divided into Y and Z.`
- `X is divided into Y and Z.`
- `X contains Y and Z.`
- `X includes Y and Z.`
- `X has N components: Y and Z.`
- `X has N parts: Y and Z.`
- `X has N sections: Y and Z.`
- `Two major components of X are Y and Z.`
- `Two basic components of X are Y and Z.`
- `Two key components of X are Y and Z.`
- `Two important components of X are Y and Z.`
- `Two essential components of X are Y and Z.`

## Evaluation Checks

- Does the expression match the selected [report genre](report.md#genre-types)?
- Does the expression preserve the report's primary organizing relation?
- Are placeholders replaced with specific, domain-appropriate content?
- Are parallel items expressed with parallel grammar?
- Does the expression avoid confusing a type with a component or a characteristic?
